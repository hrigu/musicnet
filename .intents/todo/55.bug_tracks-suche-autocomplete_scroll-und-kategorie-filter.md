# 55 - Bug (Tracks-Suche-Autocomplete): Scrollbares Dropdown und Kategorie-Filter für Vorschläge

## 1. Why

### 1.1 Objective

Die Suchvorschläge (Autocomplete) auf `/tracks` haben zwei Mängel: (1) das Vorschlags-Dropdown ist
nicht scrollbar, weiter unten stehende Einträge sind dadurch nicht erreichbar; (2) die Vorschläge
ignorieren den aktiven Anzeige-Kategorie-Filter aus den Einstellungen (Intent 54) — bei "Nur
Fusion" erscheinen trotzdem Blues-Playlists (sowie Artists/Genres, die nur in Blues-Playlists
vorkommen) als Vorschlag.

### 1.2 Context

`TrackQuerySuggestions` (Intent 43/49) liefert Werte für `playlist:`/`artist:`/`genre:`/`album:`
komplett ohne Bezug zum eingeloggten User — `TracksController#query_suggestions` ruft
`TrackQuerySuggestions.for(params[:term])` ohne `current_user` auf. Der Anzeige-Kategorie-Filter
(Intent 54, `User#active_category_substring`, Scopes `Playlist.in_active_category`/
`Track.in_active_category`/`Artist.in_active_category`) ist bereits auf `/tracks`, `/playlists`,
`/artists` verdrahtet, fehlt aber genau hier.

Das Scroll-Problem: `.search-suggestions-list` (in `tracks/index.html.erb`) nutzt nur Bootstraps
Standard-`.dropdown-menu`-Klasse, die kein `max-height`/`overflow-y` setzt — eine lange Liste
wächst unbegrenzt statt zu scrollen.

### 1.3 Depends On Intents

* 43 - Feature (Tracks-Suche): DSL-Suche mit Autocomplete
* 54 - Feature (Anzeige): Aktive Kategorie als Anzeige-Filter

## 2.0 What

```gherkin
Feature: Tracks-Suche-Autocomplete - Scrollbares Dropdown & Kategorie-Filter

  Als DJ mit aktiver Kategorie "Nur Fusion"
  möchte ich beim Tippen im Suchfeld nur Fusion-relevante Vorschläge sehen
  und bei vielen Treffern alle per Scrollen erreichen können

  Scenario: Playlist-Vorschläge respektieren die aktive Kategorie
    Given die aktive Kategorie ist "fusion"
    And es gibt eine Playlist "Blues Abend" und eine Playlist "Fusion Abend"
    When ich "playlist:a" eingebe
    Then sehe ich nur "playlist:Fusion Abend" als Vorschlag

  Scenario: Artist-Vorschläge respektieren die aktive Kategorie
    Given die aktive Kategorie ist "fusion"
    And es gibt einen Artist, der nur in einer Blues-Playlist vorkommt
    And es gibt einen Artist, der nur in einer Fusion-Playlist vorkommt
    When ich "artist:" eingebe
    Then sehe ich nur den Fusion-Artist als Vorschlag

  Scenario: Genre-Vorschläge respektieren die aktive Kategorie
    Given die aktive Kategorie ist "fusion"
    And es gibt einen Track mit Genre "Delta Blues" nur in einer Blues-Playlist
    And es gibt einen Track mit Genre "Jazz Fusion" nur in einer Fusion-Playlist
    When ich "genre:" eingebe
    Then sehe ich nur "genre:Jazz Fusion" als Vorschlag

  Scenario: Kategorie "Alle" filtert nicht
    Given die aktive Kategorie ist "all"
    And es gibt eine Playlist "Blues Abend" und eine Playlist "Fusion Abend"
    When ich "playlist:a" eingebe
    Then sehe ich beide Playlists als Vorschlag

  Scenario: Vorschlagsliste ist scrollbar
    Given es gibt mehr als eine Bildschirmhöhe an Vorschlägen
    When das Vorschlags-Dropdown offen ist
    Then hat es eine begrenzte Höhe mit eigenem Scrollbalken
```

## 3.0 How

### 3.1 Implementation Context

`TrackQuerySuggestions::VALUE_SOURCES` ist aktuell ein Hash aus Lambdas, die nur `prefix`
entgegennehmen — dafür ungeeignet, einen zusätzlichen `category_substring` einzubinden. Umbau auf
private Instanzmethoden (eine je Feld), die `@category_substring` (aus `initialize`) schliessen
können. `TrackQuerySuggestions.for` bekommt einen zweiten, optionalen Parameter
(`category_substring = nil`, Default = kein Filter, wie bei allen bestehenden
`in_active_category`-Scopes). `TracksController#query_suggestions` übergibt neu
`current_user.active_category_substring`.

Angewendete Scopes (exakt die aus Intent 54, keine neue Filterlogik):
* `playlist` → `Playlist.in_active_category(@category_substring)`
* `artist` → `Artist.in_active_category(@category_substring)`
* `genre` → `Track.in_active_category(@category_substring).distinct...pluck(:genre)`
* `album` → bleibt ungefiltert, es existiert kein `Album.in_active_category`-Scope (Intent 54 hat
  bewusst nur Playlist/Track/Artist abgedeckt) — kein neuer Scope im Rahmen dieses Bugfixes.

CSS-Fix: `.search-suggestions-list` in `app/assets/stylesheets/application.scss` (einzige
Stylesheet-Datei im Projekt) bekommt `max-height` + `overflow-y: auto`.
