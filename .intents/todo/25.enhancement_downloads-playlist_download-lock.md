# 25 - Enhancement (Downloads-Playlist): Parallele spotdl-Downloads abweisen

## 1. Why

### 1.1 Objective

Es soll nie mehr als ein spotdl-Prozess gleichzeitig laufen. Ein zweiter Download-Klick,
während einer läuft, wird mit einer verständlichen Meldung abgewiesen.

### 1.2 Context

Der Download läuft synchron im Request; der Browser zeigt lange nichts an, der Benutzer klickt
erneut — das Log zeigte drei parallele `spotdl sync`-Prozesse für dieselbe Playlist innerhalb
von 4 Sekunden. Viele parallele Requests von einer IP sind genau das Muster, das Spotifys
Rate-Limit (24h-Sperre) und YouTubes Bot-Erkennung ("You are blocked by YouTube Music")
triggert. Der `SYNC_LOCK` aus Intent 23 schützt nur den DB-Sync, nicht die Downloads.

## 2.0 What

```gherkin
Feature: Downloads Playlist - Parallele spotdl-Downloads abweisen

  Als DJ
  möchte ich dass immer nur ein spotdl-Download gleichzeitig läuft
  damit ungeduldige Doppelklicks keine parallelen Prozesse starten, die Sperren provozieren.

  Scenario: Zweiter Download wird abgewiesen
    Given ein spotdl-Download läuft
    When ich einen weiteren Download starte (Playlist-Download oder Track-Nachladen)
    Then wird er mit einer verständlichen Meldung abgewiesen
    And ich sehe die Meldung als Alert

  Scenario: Lock wird wieder freigegeben
    Given ein Download ist beendet (auch bei Fehler)
    When ich erneut einen Download starte
    Then läuft er normal an
```

## 3.0 How

### 3.1 Implementation Context

Gleiches Muster wie der `SYNC_LOCK` in `BuildMusicNetService` (Intent 23):

1. `DownloadPlaylistService` erhält `DOWNLOAD_LOCK` (Mutex) und
   `DownloadAlreadyRunningError`; `#download` läuft in `with_download_lock` (try_lock,
   Freigabe im ensure). Der Lock sitzt bewusst pro Playlist-Sync — auch der
   `DownloadTrackService`-Batch läuft dadurch spotdl-seriell, und ein konkurrierender
   Zweit-Batch scheitert beim ersten Playlist-Sync mit der Meldung.
2. `PlaylistsController#download` und `TracksController#download` fangen den Fehler und
   redirecten mit Alert (Playlist-Seite bzw. Track-Liste).
