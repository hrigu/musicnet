# 24 - Enhancement (Playlists-Playlist): Track-Anzahl auf der Index-Seite gebündelt laden

## 1. Why

### 1.1 Objective

Die Playlist-Übersicht (`playlists#index`) soll die Track-Anzahl aller Playlists mit einer
einzigen Query laden statt mit einer `COUNT`-Query pro Playlist.

### 1.2 Context

`_playlist.erb:9` ruft `playlist.tracks.count` auf — pro gerenderter Zeile eine eigene
`SELECT COUNT(*)`-Query. Bei aktuell 227 Playlists sind das 229 Queries pro Seitenaufruf
(Log: 290ms, davon 276ms Rendering mit eingestreuten Counts).

## 2.0 What

```gherkin
Feature: Playlists Playlist - Track-Anzahl gebündelt laden

  Als DJ
  möchte ich dass die Playlist-Übersicht die Track-Anzahlen in einer Query mitlädt
  damit die Seite schnell rendert und das Log kompakt bleibt.

  Scenario: Track-Anzahl ohne Einzel-Queries
    Given mehrere Playlists mit unterschiedlich vielen Tracks
    When ich die Playlist-Übersicht aufrufe
    Then zeigt jede Zeile die korrekte Track-Anzahl
    And es wird keine COUNT-Query pro Playlist ausgeführt
```

## 3.0 How

### 3.1 Implementation Context

`PlaylistsController#index` lädt die Playlists mit
`left_joins(:playlist_tracks).select("playlists.*", "COUNT(playlist_tracks.id) AS tracks_count").group("playlists.id")`
— die Anzahl kommt als Attribut `tracks_count` mit. Das Partial `_playlist.erb` verwendet
`playlist.tracks_count` statt `playlist.tracks.count`. Das Partial wird nur von der
Index-Seite gerendert (via `_playlists.erb`), ein Fallback ist nicht nötig.

Test analog Intent 22: Query-Zählung per `ActiveSupport::Notifications` — keine Query mit
`FROM "tracks"` mehr auf der Index-Seite, und die angezeigten Zahlen stimmen.
