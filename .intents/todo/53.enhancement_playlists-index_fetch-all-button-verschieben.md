# 53 - Enhancement (Playlists-Index): "Fetch all Playlists!" ins Playlists-Menü verschieben

## 1. Why

### 1.1 Objective

Der Menüpunkt "Fetch all Playlists!" gehört inhaltlich zur Playlists-Seite, nicht ins
Hauptmenü — er soll stattdessen als Button oben auf `/playlists` erscheinen.

### 1.2 Context

Aktuell liegt der Link in `layouts/_navbar.html.erb` als eigener Menüpunkt neben Zuletzt
gespielt/Playlists/Tracks/Artists. `playlists/index.html.erb` zeigt aktuell nur eine `<h1>` ohne
Kopfzeilen-Aktion — das bestehende Muster dafür ist bereits auf `/tracks` etabliert
(`index.html.erb`: `<h1>` links, Buttons rechts in einer `d-flex justify-content-between`-Zeile).

## 2.0 What

```gherkin
Feature: Playlists-Index - Fetch all Playlists Button verschoben

  Als DJ
  möchte ich "Fetch all Playlists!" auf der Playlists-Seite statt im Hauptmenü finden
  damit das Hauptmenü aufgeräumter bleibt und die Aktion dort steht, wo sie hingehört.

  Scenario: Kein eigener Menüpunkt mehr im Hauptmenü
    When ich irgendeine Seite öffne
    Then sehe ich "Fetch all Playlists!" nicht mehr in der Navbar

  Scenario: Button oben auf der Playlists-Seite
    When ich /playlists öffne
    Then sehe ich einen Button "Fetch all Playlists!" oben auf der Seite
    And ein Klick darauf löst wie bisher den POST auf fetch_all_playlists_path aus
```

## 3.0 How

### 3.1 Implementation Context

* `layouts/_navbar.html.erb`: Listenpunkt entfernen.
* `playlists/index.html.erb`: Kopfzeile analog zu `tracks/index.html.erb` (`d-flex
  justify-content-between align-items-center`), `<h1>` links, Button rechts (`button_to` oder
  `link_to` mit `data: { turbo_method: :post }`, wie bisher).
* **Specs:** bestehende Request-Specs (`spec/requests/playlists_spec.rb`) posten bereits direkt
  auf `fetch_all_playlists_path`, unabhängig vom Link-Ort — kein Testbruch zu erwarten. Neuer Test:
  Button erscheint auf `/playlists`, nicht mehr in der Navbar.
