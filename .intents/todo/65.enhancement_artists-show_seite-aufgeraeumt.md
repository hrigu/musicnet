# 65 - Enhancement (Artists-Show): Seite aufgeräumt

## 1. Why

### 1.1 Objective

Analog zu Intent 64 (Tracks-Show) soll auch die Künstler-Detailseite (`/artists/:id`) optisch
aufgeräumt und konsistenter zum Rest der App werden: der "Alben"-Titel soll prominenter wirken, die
Alben-Tabelle soll optisch der Tracks-Tabelle gleichen, standardmässig nach Release Date sortiert
sein, "Bekanntheit" wie bei Tracks als Meter statt als nackte Zahl zeigen, und der Spotify-Link soll
als Symbol ganz hinten stehen und in einem neuen Tab öffnen.

### 1.2 Context

Aktuell (`artists/show.html.erb` + `albums/_album.erb`):
* "Alben" ist nur eine `<caption>` über der Tabelle - von Bootstrap standardmässig klein/gedämpft
  gestylt, wirkt neben dem `<h1>`-Künstlernamen unauffällig.
* Die Alben-Tabelle nutzt eine andere Klasse (`table caption-top`) als die Tracks-Tabelle
  (`table table-hover align-middle mb-0 table-tracks table-tracks-detailed` aus
  `tracks/_tracks.erb`) und wirkt dadurch stilistisch uneinheitlich.
* `Artist.albums_for_show` sortiert nicht - die Reihenfolge ist die zufällige DB-Standardreihenfolge.
* Bekanntheit (`album.popularity`) wird als nackte Zahl gezeigt, während dieselbe Kennzahl bei
  Tracks (`tracks/_track.erb`) als `track_meter`-Balken erscheint.
* Der Spotify-Link (`<a href="<%= album.url %>">zu Spotify</a>`) steht als vorletzte Spalte, zeigt
  Text statt Symbol, und öffnet im selben Tab (kein `target="_blank"`).

### 1.3 Depends On Intents

* 34 - Feature (Tracks-Index): führte `track_meter`/die Tracks-Tabellen-Klassen ein, an denen sich
  dieser Intent orientiert.
* 64 - Enhancement (Tracks-Show): gleiche Art von Aufräum-Arbeit, hier auf die Artist-Seite
  übertragen.

## 2.0 What

```gherkin
Feature: Artist-Detailseite - aufgeräumte Alben-Tabelle

  Als DJ
  möchte ich auf der Künstler-Detailseite eine übersichtliche, konsistent gestaltete Alben-Tabelle
  damit die Seite genauso aufgeräumt wirkt wie die Tracks-Seiten.

  Scenario: "Alben"-Titel ist eine prominente Überschrift
    Given ich bin auf der Detailseite eines Künstlers
    Then sehe ich "Alben" als Überschrift, nicht nur als kleine Tabellen-Caption

  Scenario: Alben standardmässig nach Release Date sortiert
    Given ein Künstler hat mehrere Alben mit unterschiedlichem Release Date
    When ich seine Detailseite besuche
    Then stehen die Alben absteigend nach Release Date sortiert (neuste zuerst)

  Scenario: Bekanntheit als Meter
    Given ein Album hat einen Bekanntheits-Wert
    When ich die Detailseite besuche
    Then sehe ich ihn als Meter-Balken, wie bei den Tracks

  Scenario: Spotify-Link als Symbol, hinten, neuer Tab
    Given ein Album hat eine Spotify-URL
    When ich die Detailseite besuche
    Then steht der Spotify-Link als letzte Spalte, als Symbol (↗) statt als Text
    And ein Klick darauf öffnet einen neuen Tab
```

## 3.0 How

### 3.1 Implementation Context

* **Titel prominenter**: `<caption>Alben</caption>` raus, stattdessen `<h2 class="mt-4">Alben</h2>`
  vor der Tabelle (kein Bootstrap-`caption`-Styling mehr, echte Überschrift wie das `<h1>` oben,
  eine Ebene kleiner).
* **Tabelle angeglichen**: `albums/_album.erb`-Tabelle bekommt dieselben Klassen wie
  `tracks/_tracks.erb` (`table table-hover align-middle mb-0 table-tracks table-tracks-detailed`).
* **Sortierung**: `Artist.albums_for_show` sortiert per `.order(release_date: :desc)` (neuste
  zuerst) - statischer Default, kein interaktiver Sort-Link wie bei Tracks (dafür gibt es hier
  keinen Bedarf, die Anzahl Alben pro Künstler ist klein).
* **Bekanntheit als Meter**: `track_meter(album.popularity, variant: "info")` statt
  `<%= album.popularity %>` - derselbe Helper, dieselbe Variante wie in `tracks/_track.erb`.
* **Spotify-Link**: als letzte Spalte verschoben, Symbol `↗` statt Text "zu Spotify",
  `target: "_blank", rel: "noopener"` (neuer Tab, ohne die alte Seite über `window.opener`
  erreichbar zu machen).
* Keine Controller-/Preload-Änderungen nötig ausser der neuen `.order` in
  `Artist.albums_for_show`.
* **Tests**: Request-Specs auf `artists#show` für Sortierung (mehrere Alben, unterschiedliches
  Release Date, prüft Reihenfolge), Bekanntheit als Meter (Progress-Bar-Element statt nackter
  Zahl), Spotify-Link (Symbol statt Text, `target="_blank"`, letzte Spalte), "Alben" als
  `<h2>`-Überschrift.
