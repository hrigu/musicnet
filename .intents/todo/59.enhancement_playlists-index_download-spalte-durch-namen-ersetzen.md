# 59 - Enhancement (Playlists-Index): Download-Spalte durch vollen Namen ersetzen

## 1. Why

### 1.1 Objective

Die "Download"-Spalte auf `/playlists` (Link, der `download_playlist_path` per POST auslöst) wird
nicht mehr gebraucht. Sie war bisher aber die einzige Stelle, die den vollständigen Playlist-Namen
zeigt — Spalte 1 zeigt nur ein abgekürztes, farbcodiertes Badge (`playlist_short_name`). Ersatz:
eine Klartext-Namensspalte, damit der volle Name nicht verschwindet.

### 1.2 Context

`app/views/playlists/_playlist.erb`: `<td><%= link_to(playlist.name, download_playlist_path(...),
data: { turbo_method: :post }) %></td>` — Download bleibt über die Playlist-Detailseite weiterhin
erreichbar (`playlists#show`, "Download Tracks"-Button), geht also nicht verloren, nur der
Schnellzugriff direkt aus der Liste entfällt.

## 2.0 What

```gherkin
Feature: Playlists-Index - Download-Spalte durch Namen ersetzen

  Als DJ
  möchte ich auf der Playlists-Seite den vollen Namen statt eines Download-Links sehen
  damit ich Playlists auf einen Blick erkenne, ohne dass der Name verloren geht

  Scenario: Playlists-Index zeigt den vollen Namen statt eines Download-Links
    Given es gibt eine Playlist "RSpec Fusion Abende"
    When ich die Playlists-Seite aufrufe
    Then sehe ich "RSpec Fusion Abende" als Klartext
    And ich sehe keinen Download-Link mehr in der Zeile
```

## 3.0 How

### 3.1 Implementation Context

`_playlists.erb`: `<th>Download</th>` → `<th>Playlist-Name</th>` (unterscheidbar vom bereits
bestehenden `<th>Name</th>` für das Badge). `_playlist.erb`: Download-Link-`<td>` wird zu
`<td><%= playlist.name %></td>`, keine neue Preload-/Query-Anforderung.
