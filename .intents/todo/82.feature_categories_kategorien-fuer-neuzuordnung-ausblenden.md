# 82 - Feature (Categories): Kategorien für Neuzuordnung ausblenden

## 1. Why

### 1.1 Objective

Es gibt inzwischen sehr viele Tags über mehrere Kategorien verteilt. Beim manuellen Zuweisen
(Livesuche, Intent 79) wird das unübersichtlich, wenn Tags aus Kategorien auftauchen, die für den
aktuellen Fokus des DJs gerade nicht relevant sind. Ausgeblendete Kategorien sollen aus der
Zuweisungs-Suche verschwinden, damit sich der DJ auf die gerade relevanten Kategorien
konzentrieren kann - die Kategorie und ihre Tags bleiben aber vollständig erhalten, es ändert sich
nur, was beim *Neu*-Zuweisen vorgeschlagen wird.

### 1.2 Context

Diese Sichtbarkeit betrifft ausdrücklich nur *neue* Zuordnungen (die Livesuche in
`TagsController#search`), nicht bereits bestehende `TrackTag`-Zuordnungen oder die Anzeige
zugewiesener Tags an einem Track (`tracks/show.html.erb`, `tracks/_track.erb`) - ein Track, der
schon ein Tag aus einer inzwischen ausgeblendeten Kategorie trägt, zeigt dieses Tag unverändert
weiter an. Das ist bewusst dasselbe Prinzip wie bei `Library`/`active_library` (Import- vs.
Anzeigefilter, siehe `CLAUDE.md`): ein Verwaltungs-Flag steuert nur einen bestimmten Vorgang
(hier: Vorschläge beim Neuzuweisen), nie historische Daten.

Dieses Intent ist die Grundlage für Intent 83 (Inline-Tag-Zuweisung auf `/tracks`), dessen
Vorschlagsliste dieselbe Suche wiederverwendet und daher automatisch von diesem Flag profitiert -
es ist aber auch für sich allein nutzbar (wirkt sofort auf die bestehende Zuweisung auf der
Track-Detailseite).

## 2.0 What

```gherkin
Feature: Kategorien für Neuzuordnung ausblenden

  Als DJ
  möchte ich einzelne Kategorien von der Tag-Zuweisungs-Suche ausblenden können
  damit ich mich beim Zuweisen auf die aktuell relevanten Kategorien konzentrieren kann, ohne von
  der grossen Gesamtmenge an Tags abgelenkt zu werden.

  Scenario: Kategorie ausblenden
    Given eine Kategorie ist aktuell sichtbar
    When ich sie in der Kategorien-Verwaltung als "ausgeblendet" markiere und speichere
    Then tauchen ihre Tags nicht mehr in der Zuweisungs-Suche auf

  Scenario: Kategorie wieder einblenden
    Given eine Kategorie ist ausgeblendet
    When ich die Markierung entferne und speichere
    Then tauchen ihre Tags wieder in der Zuweisungs-Suche auf

  Scenario: Bestehende Zuordnungen bleiben unangetastet
    Given ein Track trägt bereits ein Tag aus Kategorie X
    When Kategorie X danach ausgeblendet wird
    Then bleibt das Tag am Track weiterhin sichtbar und unverändert

  Scenario: Übersicht in der Kategorien-Verwaltung
    Given es gibt sowohl sichtbare als auch ausgeblendete Kategorien
    When ich die Kategorien-Seite öffne
    Then erkenne ich auf einen Blick, welche Kategorien aktuell ausgeblendet sind
```

## 3.0 How

### 3.1 Implementation Context

* **Migration**: `categories.hidden_for_assignment` (boolean, `null: false, default: false`) -
  Name bewusst nicht `visible`/`active`, um Default `false` (= sichtbar, der unauffälligere/
  häufigere Fall) zu ermöglichen, ohne die Spalten-Semantik zu verdrehen.
* **`Category`**: Scope `visible_for_assignment` (`where(hidden_for_assignment: false)`).
* **`TagsController#search`**: zusätzliches `.joins(:category).merge(Category.visible_for_assignment)`
  in der bestehenden Query - einzige Stelle, die für alle Zuweisungs-Flows (Track-Detailseite,
  künftig Intent 83) gilt, da beide dieselbe Suche verwenden.
* **`categories_controller.rb`/`_form.html.erb`/`index.html.erb`**: Checkbox "Für Neuzuordnung
  ausblenden" im Formular, `hidden_for_assignment` zu `category_params` ergänzt; in der Übersicht
  ein dezentes Badge/Label bei ausgeblendeten Kategorien.
