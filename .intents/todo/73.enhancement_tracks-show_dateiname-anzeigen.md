# 73 - Enhancement (Tracks-Show): Dateiname anzeigen

## 1. Why

### 1.1 Objective

Auf der Track-Detailseite (`/tracks/:id`) soll der lokal gespeicherte Dateiname (`Track#file_name`, Intent 72) sichtbar sein.

### 1.2 Context

Intent 72 führte `Track#file_name` als DB-Spalte ein, bisher aber rein intern für die Pfad-Auflösung genutzt — nirgends in der UI angezeigt. Der DJ möchte den Dateinamen direkt auf der Detailseite sehen (z.B. um ihn in Finder/Mixxx wiederzufinden oder zu prüfen, ob Download/Backfill funktioniert haben).

### 1.3 Depends On Intents

* 72 - Enhancement (Tracks-Track): führte `Track#file_name` ein.
* 64 - Enhancement (Tracks-Show): aktuelles Seitenlayout mit der Stat-Zeile (Dauer/Genre/Energie/Tempo), in die sich der neue Wert einreiht.

## 2.0 What

```gherkin
Feature: Tracks-Show - Dateiname anzeigen

  Als DJ
  möchte ich auf der Track-Detailseite den gespeicherten Dateinamen sehen
  damit ich weiss, welche Datei lokal zu diesem Track gehört.

  Scenario: Dateiname sichtbar für heruntergeladene Tracks
    Given ein Track hat eine gesetzte file_name
    When ich seine Detailseite besuche
    Then sehe ich den Dateinamen als eigenes Feld

  Scenario: Kein Dateiname-Feld ohne gespeicherten Namen
    Given ein Track hat keine file_name (nicht heruntergeladen oder noch nicht befüllt)
    When ich seine Detailseite besuche
    Then sehe ich kein Dateiname-Feld (kein Platzhalter)
```

## 3.0 How

### 3.1 Implementation Context

* **`tracks/show.html.erb`**: neuer Block in der bestehenden `d-flex flex-wrap gap-4`-Stat-Zeile (Dauer/Genre/Energie/Tempo), bedingt gerendert wie Energie/Tempo (`if @track.file_name.present?`) — Label "Datei" + `@track.file_name` als reiner Text (kein Link).
* **Tests**: Request-Spec für `tracks#show` — Dateiname erscheint bei gesetztem `file_name`, fehlt (kein Markup) wenn `file_name` blank.
