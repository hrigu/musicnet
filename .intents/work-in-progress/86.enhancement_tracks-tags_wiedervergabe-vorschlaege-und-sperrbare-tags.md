# 86 - Enhancement (Tracks-Tags): Wiedervergabe-Vorschlaege und sperrbare Tags

## 1. Why

### 1.1 Objective

Die manuelle Tag-Zuweisung ist vorhanden, aber bei wiederholten Arbeitsmustern noch zu langsam:
nach dem Vergeben eines Tags ist die Chance gross, dass kurz darauf dasselbe Tag erneut vergeben
wird. Gleichzeitig gibt es Tags, die historisch existieren duerfen, aber nicht mehr neu
zugewiesen werden sollen. Die Zuweisung soll deshalb schneller und kontrollierter werden.

### 1.2 Context

Der bestehende Flow ist schon gut vorbereitet:

* `TagsController#search` liefert die Vorschlaege fuer beide Zuweisungs-Flows:
  * Track-Detailseite aus Intent 79
  * Inline-Zuweisung in `/tracks` aus Intent 83
* Kategorien koennen bereits fuer Neuzuordnung ausgeblendet werden (Intent 82)
* `TrackTagsController#create` ist die zentrale Stelle fuer neue manuelle Zuordnungen

Darauf soll die Aenderung aufsetzen, statt einen zweiten Vorschlagsmechanismus daneben zu bauen.

### 1.3 Depends On Intents

* 79 - Feature (Tracks): manuelle Tag-Zuweisung auf der Track-Detailseite
* 82 - Feature (Categories): Kategorien fuer Neuzuordnung ausblenden
* 83 - Feature (Tracks-Tags): bestehendes Tag inline in der Tracks-Liste zuweisen

## 2.0 What

```gherkin
Feature: Wiedervergabe-Vorschlaege und sperrbare Tags

  Als DJ
  moechte ich bei der manuellen Tag-Zuweisung meine zuletzt verwendeten Tags schnell wiederfinden
  und bestimmte Tags fuer neue Vergaben sperren koennen
  damit die taegliche Verschlagwortung schneller und konsistenter wird.

  Scenario: Letzte Tags als Vorschlaege auf der Track-Detailseite
    Given ich habe manuell bereits mehrere Tags vergeben
    When ich das "Tag hinzufuegen"-Widget auf der Track-Detailseite oeffne
    Then sehe ich eine Liste meiner letzten verwendeten, weiterhin erlaubten Tags als Vorschlaege

  Scenario: Letzte Tags als Vorschlaege in der Tracks-Liste
    Given ich habe manuell bereits mehrere Tags vergeben
    When ich das Inline-Tag-Widget in der Tracks-Liste oeffne
    Then sehe ich dieselben letzten erlaubten Tags als Vorschlaege

  Scenario: Neues Tag erscheint danach in den letzten Vorschlaegen
    Given ich vergebe manuell ein Tag an einen Track
    When ich direkt danach bei einem anderen Track wieder ein Tag vergebe
    Then erscheint das eben vergebene Tag in den letzten Vorschlaegen

  Scenario: Bereits zugewiesene Tags werden nicht nochmals vorgeschlagen
    Given ein Track traegt ein bestimmtes Tag bereits
    When ich die Vorschlagsliste fuer diesen Track oeffne
    Then erscheint dieses Tag dort nicht

  Scenario: Gesperrtes Tag verschwindet aus Vorschlaegen und Suche
    Given ein Tag ist fuer neue Vergaben gesperrt
    When ich ein Tag manuell zuweisen will
    Then erscheint dieses Tag weder in den letzten Vorschlaegen noch in den Suchtreffern

  Scenario: Gesperrtes Tag kann nicht mehr neu zugewiesen werden
    Given ein Tag ist fuer neue Vergaben gesperrt
    When ich versuche es trotzdem manuell zuzuweisen
    Then wird keine neue TrackTag-Zuordnung erstellt
    And ich erhalte eine verstaendliche Fehlermeldung

  Scenario: Bestehende Zuordnungen bleiben bestehen
    Given ein Track traegt bereits ein inzwischen gesperrtes Tag
    When ich die Track-Seite oder Tracks-Liste oeffne
    Then bleibt das bestehende Tag sichtbar und unveraendert
```

## 3.0 How

### 3.1 Implementation Context

Design-Entscheide:

* Vorschlaege: letzte `n` manuell vergebene Tags des aktuellen Users
* Sperre: direkt am `Tag` als Attribut, nicht ueber freie Namenslisten in Config
* Wirkung der Sperre:
  * nicht mehr vorschlagen
  * nicht mehr neu zuweisen
  * bestehende Zuordnungen unveraendert anzeigen

Technischer Vorschlag:

* neue DB-Spalte auf `tags`, z.B. `assignable:boolean`, `null: false`, `default: true`
* neue Tabelle fuer den Verlauf manueller Vergaben, z.B. `tag_assignments`
  * `user_id`
  * `tag_id`
  * `created_at`
* bei erfolgreichem `TrackTagsController#create`:
  * Verlaufseintrag fuer `current_user` und `tag` schreiben
* neuer Query-Pfad fuer Vorschlaege:
  * letzte `n` eindeutige Tags des aktuellen Users
  * nur `assignable: true`
  * nur Kategorien aus `Category.visible_for_assignment`
  * optional ohne bereits zugewiesene Tags des aktuellen Tracks
* `TagsController#search` zusaetzlich auf `assignable` einschraenken
* UI:
  * auf Track-Detailseite unter/ueber dem Suchfeld kleine Vorschlagsliste
  * im Inline-Widget in `/tracks` dieselbe Vorschlagsliste
* Admin-UI:
  * Checkbox im Tag-Formular, z.B. "Fuer neue Vergaben sperren"
  * Markierung in der Kategorien-/Tag-Uebersicht
