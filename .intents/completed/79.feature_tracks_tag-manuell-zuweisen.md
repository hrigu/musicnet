# 79 - Feature (Tracks): Tag manuell einem Track zuweisen

## 1. Why

### 1.1 Objective

Die automatische Verschlagwortung (Intent 77/78) deckt nur ab, was aus Playlist-Namen ableitbar
ist. Der DJ wollte Tags auch manuell auf der Track-Detailseite setzen können: entweder einen
bestehenden Tag (per Livesuche, inkl. Kategorie) oder einen neuen (mit Kategorie-Auswahl als
Zwischenschritt), danach die Stärke - inklusive Abbruchmöglichkeit, wenn man beim zweiten Schritt
merkt, das falsche Tag erwischt zu haben, kompletter Tastaturbedienbarkeit ohne Maus, und der
Möglichkeit, eine Zuordnung wieder zu entfernen.

### 1.2 Context

Alle Zwischenschritte (Tag-Auswahl/-Neuanlage → ggf. Kategorie → Stärke) sollten clientseitig
ablaufen, mit nur einem einzigen Server-Roundtrip am Ende, statt eines Requests pro Schritt -
konsistent mit dem bestehenden Autocomplete-Muster
(`search_suggestions_controller.js`/`TrackQuerySuggestions`) und der allgemeinen
Hotwire/Stimulus-Philosophie dieser App (minimal JS, kein SPA-Framework).

### 1.3 Depends On Intents

* 77 - Feature (Tags): liefert `Category`/`Tag`/`TrackTag`, wiederverwendet für die manuelle
  Zuordnung.
* 43 - Feature (Tracks-Suche): `search_suggestions_controller.js` als Vorbild für
  debounced-Fetch-Dropdown-Muster.

## 2.0 What

```gherkin
Feature: Tag manuell einem Track zuweisen

  Als DJ
  möchte ich auf der Track-Detailseite einen bestehenden oder neuen Tag mit Kategorie und Stärke
  zuweisen oder eine Zuordnung wieder entfernen können, komplett ohne Maus bedienbar
  damit ich Tracks auch dann verschlagworten kann, wenn die Playlist-Namen es nicht hergeben.

  Scenario: Bestehenden Tag zuweisen
    Given ich tippe einen Namen, der zu einem bestehenden Tag passt
    When ich ihn aus der Livesuche wähle und eine Stärke setze
    Then ist der Track mit diesem Tag und dieser Stärke verknüpft

  Scenario: Neuen Tag anlegen
    Given ich tippe einen Namen ohne bestehenden Treffer
    When ich "Neuer Tag" wähle, eine Kategorie auswähle und eine Stärke setze
    Then wird ein neuer Tag in dieser Kategorie angelegt und mit dem Track verknüpft

  Scenario: Kein Duplikat bei zufällig gleichem Namen
    Given der eingegebene "neue" Name existiert in der gewählten Kategorie bereits
    Then wird der bestehende Tag verwendet statt eines Duplikats

  Scenario: Abbruch bei falscher Auswahl
    Given ich habe versehentlich das falsche Tag gewählt und merke es beim Stärke- oder
    Kategorie-Schritt
    When ich "Zurück" klicke
    Then komme ich zur Suche zurück, ohne dass die falsche Auswahl übernommen wird

  Scenario: Komplett per Tastatur
    Given ich tippe einen Suchbegriff
    When ich mit Pfeiltasten/Enter navigiere statt zu klicken
    Then kann ich den gesamten Ablauf (Auswahl, Kategorie, Stärke, Abschicken) ohne Maus
    abschliessen

  Scenario: Zuordnung entfernen
    Given ein Track hat einen zugewiesenen Tag
    When ich ihn über den "×"-Button entferne und bestätige
    Then ist die Zuordnung weg, der Tag selbst bleibt aber bestehen
```

## 3.0 How

### 3.1 Implementation Context

* **`TagsController#search`**: JSON-Livesuche (`GET /tags/search?term=...`), liefert Name +
  Kategorie pro Treffer.
* **`TrackTagsController#create`/`#destroy`**: `resolve_tag` unterscheidet bestehenden Tag
  (`tag_id`) von neuem (`tag_name` + `category_id`, via `find_or_create_by!` dedupliziert).
  `find_or_initialize_by(tag:)` beim Erstellen erlaubt, die Stärke eines bereits verknüpften Tags
  ohne Fehler zu aktualisieren, statt eine Uniqueness-Verletzung zu werfen.
* **`app/javascript/controllers/tag_assign_controller.js`**: rein clientseitige Schritte
  (Suche → ggf. Kategorie → Stärke), ein Submit am Ende. Pfeiltasten/Enter navigieren die
  Vorschlagsliste (`onSearchKeydown`, erster Treffer automatisch vormarkiert, letzter Eintrag ist
  immer "Neuer Tag"), `backToSearch` setzt bei Bedarf zurück auf den Such-Schritt.
* **View**: "+ Tag hinzufügen"-Button + Formular-Panel auf `tracks/show.html.erb`, "×"-Button pro
  Tag mit `turbo_confirm` fürs Entfernen (analog zum bestehenden Playlist-Entfernen-Button).
