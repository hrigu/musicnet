# 46 - Feature (Help-Suche-Syntax): Hilfeartikel zur DSL-Suche

## 1. Why

### 1.1 Objective

Ein kleiner Hilfeartikel zur DSL-Suche (Intent 43/45) soll als Markdown-Datei in `doc/` existieren
und zusätzlich über einen neuen Menüpunkt im GUI abrufbar sein — inklusive des Hinweises, dass
zwischen `feld:` und dem Wert kein Leerzeichen stehen darf (siehe Chat-Diskussion: `artist: James
Cotton` findet nichts, `artist:"James Cotton"` funktioniert).

### 1.2 Context

Aktuell gibt es nur den knappen Inline-Hinweis im Suchformular (`index.html.erb`,
`.search-syntax-hint`, Intent 43) — keine ausführlichere Erklärung und keine eigene Seite. `doc/`
enthält bisher nur `diary.md` (Entwickler-Tagebuch) und ein Bild, keine nutzerorientierte Doku.

### 1.3 Depends On Intents

* 43 - Feature (Tracks-Index): DSL-basierte Textsuche — Gegenstand des Hilfeartikels.
* 45 - Enhancement (Tracks-Index): Suchfeld verbreitern, Verfügbarkeits-Filter entfernen.

## 2.0 What

```gherkin
Feature: Hilfeartikel zur DSL-Suche

  Als DJ
  möchte ich die Syntax der DSL-Suche nachschlagen können
  damit ich mich nicht auf den knappen Inline-Hinweis verlassen muss.

  Scenario: Hilfeartikel existiert als Markdown-Datei
    Given das Repository
    When ich in doc/ nachsehe
    Then existiert eine Markdown-Datei mit allen Feldern, Operatoren, Beispielen und dem
      Leerzeichen-Hinweis

  Scenario: Hilfeartikel ist über einen Menüpunkt erreichbar
    Given ich bin eingeloggt
    When ich auf den neuen Menüpunkt klicke
    Then sehe ich den Hilfeartikel als HTML-Seite

  Scenario: Eine Quelle für den Inhalt
    Given der Hilfeartikel wird angepasst
    When nur die Markdown-Datei in doc/ geändert wird
    Then zeigt die GUI-Seite automatisch den neuen Inhalt (keine zweite, separat zu pflegende Kopie)
```

## 3.0 How

### 3.1 Implementation Context

* **Gem:** `redcarpet` (schlank, keine Laufzeit-Abhängigkeiten, C-Extension, weit verbreitet) für
  Markdown → HTML zur Laufzeit.
* **Inhalt:** `doc/track_search_syntax.md` — Feldliste (`artist`, `album`, `genre`, `playlist`,
  `bpm`/`tempo`, `energy`, `popularity`, `year`/`release`), Komma = ODER, `min..max`-Range,
  Vergleichsoperatoren, `-`-Negation, Anführungszeichen für Werte mit Leerzeichen, Wiederholung
  desselben Feldes = UND (Playlist-Schnittmenge), und explizit der Hinweis: kein Leerzeichen
  zwischen `feld:` und dem Wert (Beispiel: `artist:"James Cotton"` statt `artist: James Cotton`).
* **Route/Controller:** neuer `HelpController#search_syntax` (`GET /help/suche-syntax`), liest
  `doc/track_search_syntax.md` und rendert es via `Redcarpet::Markdown.new(Redcarpet::Render::HTML)
  .render(...)` in eine einfache View (Bootstrap-Grundstyling, keine neue Layout-Variante nötig).
* **Navbar:** neues Dropdown "Hilfe" in `layouts/_navbar.html.erb` (Bootstrap
  `dropdown`/`dropdown-menu`, analog zu bestehenden Bootstrap-Komponenten im Projekt) mit einem
  ersten Eintrag "Suche", der auf die neue Route verlinkt — bewusst als Dropdown statt flachem
  Link, damit künftige weitere Hilfeartikel als weitere Einträge dazukommen können, ohne die
  Navbar-Struktur nochmals zu ändern.
* **Specs:** Request-Spec für `HelpController#search_syntax` (Status 200, enthält z.B. "playlist:"
  und den Leerzeichen-Hinweis-Text im gerenderten HTML).
