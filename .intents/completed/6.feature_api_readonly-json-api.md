# 6 - Feature (Api): Read-only JSON-API mit Bearer-Token-Auth

> Rückwirkend aus Code und Git-Historie rekonstruiert (nicht vor der Umsetzung geschrieben).
> **Datum (ungefähr):** September 2023 (erste Version), Oktober 2023 (Rückbau von graphiti zugunsten der schlanken Bearer-Token-API).

## Why

Externe Clients (z.B. andere eigene Tools) sollen lesend auf die lokal gespiegelten Playlist-Daten
zugreifen können, ohne die Devise-Session der Haupt-App zu nutzen.

## What

- `/api/v1/*` liefert JSON und ist per statischem Bearer-Token geschützt, unabhängig vom
  Devise-Login der übrigen App.
- Playlists können gelistet und einzeln abgerufen werden.

## How

- [x] `ApiToken`-Modell + `Api::V1::BaseController#authenticate_user_with_token`: prüft aktive
      Tokens gegen den `Authorization: Bearer`-Header.
- [x] `Api::V1::PlaylistsController#index/show`: JSON-Rendering der lokalen `Playlist`-Daten.
- [x] `Api::V1::HomeController#index` als einfacher Health-/Info-Endpoint.
- [x] Frühere `graphiti`-basierte API-Implementierung (`api/v2`) wieder zurückgebaut (siehe Commit
      "API Rückgebaut") zugunsten der schlankeren Bearer-Token-Lösung.
