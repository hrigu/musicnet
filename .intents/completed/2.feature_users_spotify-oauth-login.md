# 2 - Feature (Users): Login via Spotify OAuth

> Rückwirkend aus Code und Git-Historie rekonstruiert (nicht vor der Umsetzung geschrieben).
> **Datum (ungefähr):** Februar 2021.

## Why

Die App agiert im Namen des DJs gegenüber der Spotify Web API (eigene Playlists holen, recently
played, etc.). Dafür braucht es einen authentifizierten Spotify-User, nicht nur die App-eigenen
Client-Credentials.

## What

- Login erfolgt via Devise + `omniauth-spotify` gegen den eigenen Spotify-Account des DJs.
- Nach erfolgreichem OAuth-Callback wird der `RSpotify::User` serialisiert und am lokalen `User`
  gespeichert, damit spätere Requests als der eingeloggte Spotify-User laufen können.

## How

- [x] Devise-Setup mit `omniauth-spotify`-Strategy (`config/initializers/devise.rb`), inkl. der
      benötigten Scopes.
- [x] `omniauth-rails_csrf_protection` eingebunden (ohne das wirft omniauth
      `OmniAuth::AuthenticityTokenProtection`).
- [x] `UsersController#spotify` als OAuth-Callback, speichert `spotify_user_data` (JSON) auf `User`.
- [x] `User#spotify_user` rekonstruiert daraus ein `RSpotify::User`-Objekt für spätere API-Calls.
