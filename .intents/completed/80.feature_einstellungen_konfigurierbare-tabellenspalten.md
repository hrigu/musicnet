# 80 - Feature (Einstellungen): Konfigurierbare Spalten in Tracks-/Playlist-Tabellen

## 1. Why

### 1.1 Objective

Nachdem Tags (Intent 77) viel von dem abdecken, was bisher nur über die Playlists-Spalte sichtbar
war, stellte sich die Frage, ob diese Spalte noch gebraucht wird. Nach Abwägung (Playlists =
exakte, operative Zugehörigkeit inkl. Verwaltungslink; Tags = verlustbehaftete semantische
Verdichtung davon, u.a. ohne Tag-Treffer für ~12 Playlists) entschied sich der DJ nicht für ein
Entfernen, sondern dafür, alle Spalten der Tracks-/Playlist-Tabellen individuell ein-/ausblendbar
zu machen, statt nur einen Playlists-Sonderfall zu bauen.

### 1.2 Context

Die bestehende CSS-Struktur (`app/assets/stylesheets/application.scss`,
`.table-tracks-detailed th:nth-child(N)`) band Spaltenbreiten an ihre feste Position - eine
bedingt gerenderte (ausgeblendete) Spalte hätte alle nachfolgenden `nth-child`-Positionen
verschoben und falsche Breiten erzeugt. Diese Struktur musste zuerst auf klassenbasierte
Selektoren (`.col-name`, `.col-genre`, ...) umgestellt werden, unabhängig von der Position im DOM.

### 1.3 Depends On Intents

* 77 - Feature (Tags): Auslöser der Diskussion (Tags vs. Playlists-Spalte), lieferte die
  "Tags"-Spalte, die jetzt Teil der konfigurierbaren Spaltenliste ist.
* 57 - Feature (Bibliotheken): `SettingsController`/`settings/edit.html.erb` als Ort für die neue
  Einstellung, analog zur bestehenden `active_library`-Auswahl.

## 2.0 What

```gherkin
Feature: Konfigurierbare Spalten in Tracks-/Playlist-Tabellen

  Als DJ
  möchte ich in den Einstellungen jede Spalte der Tracks-/Playlist-Tabellen einzeln ein- oder
  ausblenden können
  damit ich die Ansicht auf das reduzieren kann, was mich gerade interessiert.

  Scenario: Spalte ausblenden
    Given ich hake in den Einstellungen "Playlists" ab (entferne den Haken)
    When ich speichere
    Then verschwindet die Playlists-Spalte auf /tracks und in der Playlist-Detailansicht

  Scenario: Andere Einstellung ändert nichts an den Spalten
    Given ich habe zuvor eine Spalte ausgeblendet
    When ich nur die aktive Bibliothek ändere (ohne die Spalten-Checkboxen erneut abzusenden)
    Then bleibt die zuvor ausgeblendete Spalte weiterhin ausgeblendet

  Scenario: Name-Spalte ist immer sichtbar
    Given ich befinde mich auf der Einstellungen-Seite
    Then gibt es keine Checkbox, um die Name-Spalte auszublenden
```

## 3.0 How

### 3.1 Implementation Context

* **Migration**: `users.hidden_track_columns` (JSON, Default `[]`) - speichert die AUSGEBLENDETEN
  Spalten-Keys, nicht die sichtbaren, damit ein User ohne diese Einstellung (leeres Array)
  unverändert alle Spalten sieht wie vor Einführung dieser Funktion.
* **`Track::OPTIONAL_COLUMNS`**: zentrale, geordnete Liste der 12 ausblendbaren Spalten-Keys +
  Labels (Name fehlt bewusst, da sie den Zeilenlink trägt und nie ausblendbar ist).
* **`User#column_visible?`** + `TracksHelper#column_visible?` (Delegation).
* **`SettingsController`**: Checkboxen senden nur angehakte (= sichtbare) Spalten
  (`visible_track_columns`), umgerechnet auf die gespeicherten `hidden_track_columns`. Ein
  führendes leeres Hidden-Field verhindert, dass "keine Checkbox angehakt" als fehlender Parameter
  statt als leeres Array ankommt. **Wichtiger, beim Bauen gefundener Bug**: `user_update_params`
  darf `hidden_track_columns` nur anfassen, wenn `visible_track_columns` im Request überhaupt
  vorhanden war - sonst hätte ein Request, der nur `active_library_id` ändert, alle Spalten
  unbeabsichtigt ausgeblendet.
* **CSS-Umbau**: `.table-tracks-detailed th:nth-child(N)` → `.col-<key>`-Klassen. Fehlt eine
  Spalte, verteilt `table-layout: fixed` die nicht mehr zu 100% summierenden Breiten automatisch
  proportional auf die verbleibenden Spalten - keine Sonderfälle pro Spalten-Kombination nötig.
* **Views**: `tracks/_tracks.erb`/`_track.erb`, `playlists/show.html.erb`/
  `playlist_tracks/_playlist_track.erb` bekommen pro optionaler Spalte ein `<% if
  column_visible?("key") %>` um `<th>`/`<td>`, in identischer Reihenfolge zwischen Kopf- und
  Datenzeile gehalten.
