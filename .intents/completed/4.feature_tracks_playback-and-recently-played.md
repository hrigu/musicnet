# 4 - Feature (Tracks): Track-Wiedergabe und Recently-Played-Anzeige

> Rückwirkend aus Code und Git-Historie rekonstruiert (nicht vor der Umsetzung geschrieben).
> **Datum (ungefähr):** August bis Oktober 2023.

## Why

Vor dem Download/Auflegen will der DJ prüfen können, was er zuletzt auf Spotify gehört hat, und
bereits heruntergeladene Tracks lokal probehören können.

## What

- Startseite zeigt die letzten 50 auf Spotify gespielten Tracks (live via Spotify API, nicht
  lokal gespiegelt).
- Lokal heruntergeladene Tracks können direkt im Browser als Audiofile gestreamt werden.
- Ein direktes Abspielen via Spotify-Player-API wurde versucht, funktioniert aber nicht
  (`RestClient::Unauthorized`) und ist als bekannter, nicht behobener Defekt im Code dokumentiert.

## How

- [x] `TracksController#recently_played_index` (root route) über `current_user.spotify_user.
      recently_played(limit: 50)`.
- [x] `TracksController#stream`: liefert die lokale Datei über `send_file` anhand `Track#track_path`.
- [x] `TracksController#index/show`: Übersicht/Detail der lokal gespiegelten Tracks.
- [~] `TracksController#play`: Versuch, direkt in Spotify abzuspielen — bekanntermassen defekt,
      nicht Teil des funktionierenden Funktionsumfangs.
