# 9 - Test (Downloads): DownloadTrackService testen

## Why

Analog zu `DownloadPlaylistService` — baut das `spotdl`-Kommando für einzelne Tracks, ungetestet.

## What

Testabdeckung für `#download` — korrektes spotdl-download-Kommando mit einem oder mehreren
Track-URLs (aus `spotify_id`), Verzeichniswechsel. `system`/`Dir.chdir` wieder gestubbt.

## How

TDD, ein Szenario nach dem anderen, `bundle exec rspec spec/services/download_track_service_spec.rb`
nach jedem Schritt.

* [x] 1.0 - Testsuite für `DownloadTrackService` aufbauen:
    - [x] 1.1 - Ruft `system` mit dem korrekten spotdl-download-Kommando für einen Track auf
    - [x] 1.2 - Fügt mehrere Track-URLs im Kommando leerzeichengetrennt zusammen
    - [x] 1.3 - Wechselt vorher ins `downloads/tracks`-Verzeichnis
