# 26 - Bug (Playlists-Playlist): tracks_count fehlt ausserhalb des Playlist-Index

## 1. Why

### 1.1 Objective

Die Track-Detailseite (`tracks#show`) soll wieder rendern. Sie zeigt die Playlists des Tracks
über das Partial `playlists/_playlist.erb`, das seit Intent 24 `playlist.tracks_count`
erwartet — das Attribut existiert aber nur, wenn die Playlists mit dem COUNT-Select des
Playlist-Index geladen wurden. Ergebnis: `undefined method 'tracks_count'` auf `/tracks/:id`.

### 1.2 Context

Die Annahme in Intent 24 ("das Partial wird nur von der Index-Seite gerendert") war falsch:
`tracks/show.html.erb:67` rendert `@track.playlists` mit demselben Partial.

## 2.0 What

```gherkin
Feature: Playlists Playlist - tracks_count überall verfügbar

  Als DJ
  möchte ich die Track-Detailseite mit den Playlist-Badges sehen
  damit ich sehe, in welchen Playlists ein Track steckt.

  Scenario: Track-Detailseite rendert wieder
    Given ein Track ist in einer Playlist
    When ich die Track-Detailseite aufrufe
    Then sehe ich die Playlist-Zeile inkl. Track-Anzahl

  Scenario: Playlist-Index bleibt gebündelt
    When ich die Playlist-Übersicht aufrufe
    Then wird weiterhin keine COUNT-Query pro Playlist ausgeführt
```

## 3.0 How

### 3.1 Implementation Context

`Playlist#tracks_count` als Methode: liefert den per SELECT-Alias mitgeladenen Wert
(`self[:tracks_count]`), sonst `tracks.count` als Fallback. Der Index nutzt weiterhin die
gebündelte Query (bestehende Query-Zähler-Spec sichert das ab); alle anderen Render-Stellen
funktionieren mit dem Fallback.
