# 7 - Test (Spotify-Playlists): BuildMusicNetService testen

## Why

`BuildMusicNetService` ist das Herzstück der App (Sync + Orphan-Cleanup), aber komplett
ungetestet. Ein Fehler hier korrumpiert stillschweigend die lokale Datenbasis, auf der Download
und Mixxx-Export aufbauen.

## What

Testabdeckung für `#build`, inkl. Idempotenz, Owner-/Namensfilter und Orphan-Cleanup. Die
Spotify-Web-API ist die externe Grenze und wird gestubbt (kein webmock/VCR im Projekt vorhanden,
also über RSpotify-Objekt-Doubles).

## How

TDD, ein Szenario nach dem anderen, `bundle exec rspec spec/services/build_music_net_service_spec.rb`
nach jedem Schritt.

* [x] 1.0 - Testsuite für `BuildMusicNetService` aufbauen:
    - [x] 1.1 - Test-Helper für RSpotify-Doubles (Playlist/Track/Album/Artist) vorbereiten
    - [x] 1.2 - Playlist mit Track wird neu erstellt (inkl. Album, Artists, PlaylistTrack)
    - [x] 1.3 - Erneuter Lauf legt keine Duplikate an (Idempotenz)
    - [x] 1.4 - Playlists, die nicht dem aktuellen User gehören, werden ignoriert
    - [x] 1.5 - Playlists ohne "fusion"/"blues" im Namen werden nicht importiert
    - [x] 1.6 - Playlists ohne Tracks werden nach dem Sync gelöscht
    - [x] 1.7 - Tracks/Artists/Albums ohne Zuordnung werden nach dem Sync gelöscht
    - [x] 1.8 - `ServiceInfo` sammelt erstellte/gelöschte Namen korrekt
