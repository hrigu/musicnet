# 61 - Enhancement (Playlists-Index): Anzahl heruntergeladener Tracks anzeigen

## 1. Why

### 1.1 Objective

Die "Anzahl Tracks"-Spalte auf `/playlists` zeigt nur die Gesamtzahl. Der DJ möchte auf einen
Blick sehen, wie viele davon bereits als lokale Audiodatei vorliegen, ohne jede Playlist einzeln
zu öffnen.

### 1.2 Context

Downloaded-Status wird bereits an anderer Stelle bestimmt (`playlists#show`,
`PlaylistsHelper#all_tracks_downloaded?`) über `Track#track_path` (Datei-Existenz, per
`TrackFileLocator` ermittelt). `Track.preload_track_paths(tracks)` muss vor dem Zugriff auf viele
Tracks einmal für das ganze Batch aufgerufen werden (ein Verzeichnis-Scan statt einem pro Track,
siehe CLAUDE.md). Auf dem Index gibt es aktuell keinen Track-Preload, nur `tracks_count`
(Counter-Cache).

## 2.0 What

```gherkin
Feature: Playlists-Index - Anzahl heruntergeladener Tracks

  Als DJ
  möchte ich auf der Playlists-Seite sehen, wie viele Tracks bereits heruntergeladen sind
  damit ich auf einen Blick erkenne, welche Playlists noch Downloads brauchen

  Scenario: Anzahl-Tracks-Spalte zeigt Gesamt- und Downloaded-Anzahl
    Given eine Playlist hat 5 Tracks, davon 3 mit lokaler Audiodatei
    When ich die Playlists-Seite aufrufe
    Then sehe ich "5 (3)" in der Anzahl-Tracks-Spalte
```

## 3.0 How

### 3.1 Implementation Context

`PlaylistsController#index`: zusätzlich `.includes(:tracks)`, danach einmalig
`Track.preload_track_paths(@playlists.flat_map(&:tracks))` (ein Verzeichnis-Scan für die ganze
Seite, nicht pro Playlist/Track). Neuer Helper `PlaylistsHelper#downloaded_tracks_count(playlist)`
zählt `playlist.tracks.count { |t| t.track_path.present? }` auf der bereits preloadeten
Association. View: `<%= playlist.tracks_count %> (<%= downloaded_tracks_count(playlist) %>)`.
`tracks_count` bleibt der Counter-Cache (Gesamtzahl), nur die Downloaded-Zahl kommt aus dem neuen
Preload.
