# 45 - Enhancement (Tracks-Index): Suchfeld verbreitern, Verfügbarkeits-Filter entfernen

## 1. Why

### 1.1 Objective

Das Suchfeld auf `/tracks` ist zu schmal für die DSL-Suche (Intent 43) — typische Queries wie
`genre:jazz,fusion bpm:80..100 playlist:"Fusion Abende"` sind länger als das bisherige Feld
komfortabel darstellt. Zusätzlich soll der bestehende Verfügbarkeits-Filter (Dropdown
"Alle/Nur heruntergeladen/Nur fehlend") inklusive des zugehörigen Codes vollständig entfernt
werden.

### 1.2 Context

Das Suchfeld (`app/views/tracks/index.html.erb`) hat seit der Autocomplete-Einführung (Intent 43,
Task 6) keine explizite Breite mehr (`class: "form-control"` innerhalb eines
`d-inline-block`-Containers ohne eigene Breite) — davor war es `form-control d-inline-block
w-auto`, was sich am Browser-Default orientierte und für die DSL ebenfalls zu knapp ist.

Der Verfügbarkeits-Filter stammt aus Intent 34: `TracksController::AVAILABLE_FILTERS`,
`#paginate_by_availability`, `#filter_by_availability` (weicht für `available=downloaded/missing`
vom normalen SQL-Pagination-Pfad auf eine Ruby-Array-Filterung nach `Track#track_path` aus), das
Dropdown im Suchformular, der Describe-Block "GET /tracks - Verfügbarkeits-Filter" in
`spec/requests/tracks_spec.rb` sowie die Dokumentation dazu im CLAUDE.md-Abschnitt "Tracks index".

### 1.3 Depends On Intents

* 34 - Feature (Tracks-Index): Paginierung, Sortierung, Volltextsuche — führte den zu entfernenden
  Verfügbarkeits-Filter ein.
* 43 - Feature (Tracks-Index): DSL-basierte Textsuche — Grund für die grössere Feldbreite.

## 2.0 What

```gherkin
Feature: Tracks-Index - Suchfeld-Breite und entfernter Verfügbarkeits-Filter

  Als DJ
  möchte ich ein für die DSL ausreichend breites Suchfeld
  und keinen Verfügbarkeits-Filter mehr sehen
  damit die Suche übersichtlich bleibt und ich nicht durch eine nicht mehr benötigte Option
  abgelenkt werde.

  Scenario: Suchfeld ist breiter
    Given ich bin auf der Tracks-Seite
    When die Seite lädt
    Then hat das Suchfeld eine grössere Mindestbreite als zuvor

  Scenario: Kein Verfügbarkeits-Filter mehr sichtbar
    Given ich bin auf der Tracks-Seite
    When die Seite lädt
    Then sehe ich kein Dropdown für "Alle/Nur heruntergeladen/Nur fehlend"

  Scenario: available-Parameter hat keine Wirkung mehr
    Given ich rufe /tracks mit ?available=downloaded auf
    When die Seite lädt
    Then werden trotzdem alle Tracks normal paginiert angezeigt (Parameter wird ignoriert)

  Scenario: Suche, Sortierung und Pagination funktionieren unverändert
    Given die bestehende DSL-Suche und Sortierung
    When ich sie benutze
    Then funktionieren sie wie vor dieser Änderung
```

## 3.0 How

### 3.1 Implementation Context

* **Suchfeld-Breite:** neue SCSS-Klasse (z.B. `.search-query-input`) in `application.scss` mit
  einer `min-width` (analog zu bestehenden Klassen wie `.track-meter`), auf das Suchfeld in
  `index.html.erb` angewendet.
* **Verfügbarkeits-Filter entfernen:**
  * View: `select_tag :available, ...` aus `index.html.erb` entfernen.
  * Controller: `AVAILABLE_FILTERS`-Konstante, `#paginate_by_availability`,
    `#filter_by_availability` löschen; `#paginate_for_index` vereinfacht sich auf den bisherigen
    einzigen (SQL-)Pfad (`pagy(:offset, tracks, limit: PAGE_SIZE)` + `Track.preload_track_paths`).
  * Specs: Describe-Block "GET /tracks - Verfügbarkeits-Filter" in `spec/requests/tracks_spec.rb`
    vollständig entfernen.
  * CLAUDE.md: `available`-Bullet-Point im Abschnitt "Tracks index" entfernen.
* Kein TDD-Rot-Grün im klassischen Sinn für die Entfernung (kein neues Verhalten) — stattdessen:
  Suite vor der Änderung grün bestätigen, Code+Tests gemeinsam entfernen, Suite danach erneut
  grün bestätigen (zeigt, dass nichts anderes vom entfernten Code abhing).
