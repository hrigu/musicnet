# 10 - Test (Models): Album, Artist, Track, PlaylistTrack testen

## Why

Custom-Logik in den Models (Zeitformatierung, Audio-Feature-Parsing, Dateisystem-Zugriff,
distinct-Through-Associations) ist ungetestet. Reine AR-Assoziationen ohne Custom-Logik werden
nicht extra getestet — das garantiert Rails schon.

## What / How

TDD, ein Szenario nach dem anderen, `bundle exec rspec spec/models` nach jedem Schritt.

* [x] 1.0 - Track-Logik testen:
    - [x] 1.1 - `#dauer` formatiert `duration_ms` als MM:SS
    - [x] 1.2 - `#af`/`#energy`/`#tempo` parsen `audio_features` korrekt (inkl. nil, wenn leer)
    - [x] 1.3 - `#track_path` findet die passende Datei im `downloads/tracks`-Verzeichnis (Dateisystem gestubbt)
    - [x] 1.4 - `#track_path` gibt nil zurück, wenn keine Datei gefunden wird
    - [x] 1.5 - `#genre` gibt nil zurück, wenn kein Track-File gefunden wird (Guard Clause; WahWah-Parsing selbst wird nicht getestet)
* [x] 2.0 - Album/Artist distinct-Associations testen:
    - [x] 2.1 - `Album#artists` liefert jeden Artist nur einmal, auch bei mehreren gemeinsamen Tracks
    - [x] 2.2 - `Artist#albums` liefert jedes Album nur einmal, auch bei mehreren Tracks im selben Album
    - [x] 2.3 - `Artist#playlists_of_the_tracks` liefert alle Playlists, in denen der Artist vorkommt
* [x] 3.0 - PlaylistTrack Pflichtfelder testen:
    - [x] 3.1 - Playlist und Track sind Pflichtfelder (belongs_to-Presence-Validierung)
