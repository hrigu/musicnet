# 22 - Enhancement (Playlists-Playlist): N+1-Queries und Autoload der Audio-Files beseitigen

## 1. Why

### 1.1 Objective

Die Playlist-Detailseite soll pro Anzeige eine handvoll gebündelte SQL-Queries ausführen statt
über 100 Einzel-Queries, und die Audio-Dateien sollen erst beim Abspielen geladen werden statt
sofort beim Seitenaufbau.

### 1.2 Context

Das Log eines Aufrufs von `playlists#show` mit 20 Tracks zeigt 132 Queries (N+1) sowie einen
`/tracks/:id/stream`-Request pro Track direkt nach dem Seitenaufbau. Drei Ursachen:

1. `show.html.erb` rendert `@playlist.playlist_tracks` (frische, nicht vorgeladene Association)
   statt des im Controller per `includes` vorbereiteten `@playlist_tracks` — das Eager Loading
   läuft ins Leere.
2. Das `includes(track: { album: :artists })` passt nicht zu dem, was das Partial
   `_playlist_track.erb` tatsächlich lädt: `track.artists`, `track.album` sowie
   `track.playlist_tracks` → `pt.playlist` (Playlist-Badges).
3. `components/_audio_file.html.erb` verwendet `preload="false"` — kein gültiger HTML-Wert
   (`none|metadata|auto`). Browser fallen auf den Default zurück und laden darum jede
   Audio-Datei sofort an.

## 2.0 What

```gherkin
Feature: Playlists Playlist - N+1-Queries und Audio-Autoload beseitigen

  Als DJ
  möchte ich dass die Playlist-Seite ihre Daten gebündelt lädt
  damit die Seite schnell rendert und das Log lesbar bleibt.

  Scenario: Tracks werden gebündelt geladen
    Given eine Playlist mit mehreren Tracks inkl. Artists, Alben und Playlist-Badges
    When ich die Playlist-Seite aufrufe
    Then wird jede beteiligte Tabelle (tracks, artists, albums) mit genau einer Query geladen

  Scenario: Audio erst beim Abspielen laden
    Given eine Playlist mit Tracks
    When ich die Playlist-Seite aufrufe
    Then enthalten die Audio-Elemente preload="none"
    And es wird keine Audio-Datei automatisch angefordert
```

## 3.0 How

### 3.1 Implementation Context

Kleine, fokussierte Eingriffe — TDD über Request-Specs:

1. `show.html.erb` rendert `@playlist_tracks`; das `includes` im Controller wird auf die vom
   Partial benötigten Associations korrigiert: `track: [:artists, :album,
   { playlist_tracks: :playlist }]`. Die `refresh`-Action nutzt dieselbe Zeile — in eine
   private Methode extrahieren. Der Query-Zähler-Test nutzt
   `ActiveSupport::Notifications.subscribed` auf `sql.active_record` (keine neuen Gems).
2. `_audio_file.html.erb`: `preload="false"` → `preload="none"`.
