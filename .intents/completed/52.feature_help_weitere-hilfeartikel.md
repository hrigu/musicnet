# 52 - Feature (Help): Weitere Hilfeartikel (Installation, Bedienung, Diary)

## 1. Why

### 1.1 Objective

Das "Hilfe"-Dropdown (Intent 46, aktuell nur "Suche") soll um drei weitere Artikel erweitert
werden: eine noch zu schreibende Installationsanleitung (Inhalte aus dem README, das README soll
im Gegenzug auf diese Anleitung verlinken), eine noch zu schreibende, umfassende
Bedienungsanleitung (ursprünglicher README-Workflow **plus** die seither dazugekommenen Features:
DSL-Suche, Song-Queue, Cue-/Vorhörkanal) und das bestehende Entwickler-Diary (`doc/diary.md`).

### 1.2 Context

`HelpController` (Intent 46) hat aktuell nur eine einzige Action `#search_syntax`, fest verdrahtet
auf `doc/track_search_syntax.md`. Für vier Artikel wird das zu einer generischen,
parametrisierten Action mit einer Feld-Whitelist (Slug → Titel + Datei) — verhindert, dass ein
roher Dateiname aus der URL gelesen wird, und vermeidet vier fast identische Controller-Actions.

`README.md` enthält aktuell einen Mix aus Bedienungsworkflow ("Vorgehen"), technischen Spotify-
Infos und Setup-Hinweisen (Credentials, `master.key`/`development.key` auf neuer Maschine) — die
Setup-Hinweise werden zur neuen `doc/installation.md`, der Workflow-Teil fliesst (erweitert um
neuere Features) in die neue `doc/usage.md`.

### 1.3 Depends On Intents

* 46 - Feature (Help-Suche-Syntax): Hilfeartikel zur DSL-Suche — liefert `HelpController`,
  `redcarpet`-Rendering und das Navbar-Dropdown, die hier erweitert werden.

## 2.0 What

```gherkin
Feature: Help - Weitere Hilfeartikel

  Als DJ
  möchte ich im Hilfe-Menü auch Installationsanleitung, Bedienungsanleitung und Diary finden
  damit ich nicht zwischen README und App wechseln muss.

  Scenario: Hilfe-Dropdown zeigt vier Einträge
    Given ich bin eingeloggt
    When ich das Hilfe-Dropdown öffne
    Then sehe ich Suche, Installation, Bedienung und Diary

  Scenario: Jeder Artikel ist erreichbar
    When ich einen der vier Einträge anklicke
    Then sehe ich den jeweiligen Artikel als gerenderte HTML-Seite

  Scenario: Unbekannter Artikel-Slug
    When ich eine unbekannte Artikel-URL aufrufe
    Then bekomme ich einen 404, keinen Server-Fehler

  Scenario: README verlinkt die Installationsanleitung
    Given das README
    When ich es öffne
    Then finde ich einen Link auf doc/installation.md
```

## 3.0 How

### 3.1 Implementation Context

* **`HelpController#show`:** ersetzt `#search_syntax`. Whitelist-Konstante
  `ARTICLES = { "suche-syntax" => {title:, file:}, "installation" => {...}, "bedienung" => {...},
  "diary" => {...} }.freeze`. Unbekannter `params[:page]` → `ActionController::RoutingError`
  (404). View `app/views/help/show.html.erb` (ersetzt `search_syntax.html.erb`), zeigt `@title`
  als Überschrift.
* **Route:** `get "help/:page", to: "help#show", as: :help` ersetzt die bisherige
  `get "help/suche-syntax", to: "help#search_syntax", as: :search_syntax_help` — Korrektur
  gegenüber Intent 46: eine generische statt vier fast identischer Routen/Actions.
* **Navbar:** vier Einträge im "Hilfe"-Dropdown, `help_path(page: "...")`.
* **`doc/installation.md`** (neu): Voraussetzungen (Ruby/Rails/sqlite3, `bin/setup`), Spotify-
  Credentials (`master.key`/`development.key`, Setup auf neuer Maschine), externe Tools (`spotdl`,
  Docker + Essentia-Image), Start (`bin/rails server -p 3001`, exakte Adresse wegen OAuth-Callback).
  Quelle: README-Abschnitte "Anderes/Credentials", "Setup auf einer neuen Maschine", "Tools".
* **`doc/usage.md`** (neu, Bedienungsanleitung): Login, Playlists holen, Tracks-Suche (Verweis auf
  den Suche-Syntax-Artikel), Song-Queue, Cue-/Vorhörkanal, Download, Mixxx-Crates erstellen/
  importieren. Quelle: README-Abschnitt "Vorgehen", erweitert um die Features aus Intent 41-51.
* **README.md:** Anpassung des bestehenden Diary-Links-Musters — neuer Link auf
  `doc/installation.md` an der Stelle der bisherigen Setup-Anleitung.
* **Specs:** Request-Specs je Artikel (Status 200, enthält einen charakteristischen Text-
  Ausschnitt), 404-Test für unbekannten Slug, Navbar-Test für alle vier Einträge.
